The savings modelPure, deterministic, TDD'd domain core — this is where credibility lives

Reconstruct physical baselines (kWh, km, litres) from spend ÷ local price, then compute three buckets.

⚡ Electricity

€61/mo
PV self-consumption + feed-in of surplus + battery arbitrage (charge cheap / discharge expensive on dynamic tariff)

🔥 Heating

€55/mo
oil/gas spend → kWh heat ÷ SCOP (3.0–4.0, climate) = heat-pump electricity (partly self-generated)

🚗 Mobility

€26/mo
petrol spend → km/year → EV kWh charged at PV surplus + off-peak dynamic blend
The upgrade ladder — cumulative, not a menu
Each rung keeps everything below it and unlocks one new value stream. The optimiser walks the ladder and stops at the depth with the highest cumulative net saving — deeper isn't always better.
RungAddsUnlocks value streamBucketΔ netCumulative
①Solar / PVself-consumption of own generationElectricity (a)+€38€38
②+ Battery↑ self-consumption + dynamic-tariff arbitrageElectricity (b)+€23€61
③+ Heat pumpfossil heating → (partly self-generated) electricityHeating+€55€116
④ ★+ EV chargerpetrol → off-peak / PV chargingMobility+€26€142
Up-sell = the next rung, quantified. Solar alone saves €38/mo; because the home still burns oil at €180/mo, the heat-pump rung adds €55 — the biggest single jump. Each rung carries its own capex → own installment, so Δ net = Δ gross − Δ installment.
Why cumulative ≠ sum of independents: later rungs raise household electricity demand, which increases the value of the PV + battery already installed (more solar self-consumed, battery cycles more usefully, flexible EV/heat-pump load pairs with the dynamic tariff). The engine computes each rung on top of the running state, never in isolation.
💶 Financing overlay (the anchor)
capex_after_subsidy = capex − subsidies installment = annuity(financed, apr, tenor) net_now = gross_saving − installment net_after = gross_saving (loan gone) → report both + break-even month
🎯 Certainty (no false precision)
Each default carries an uncertainty → propagate to a band (±€Y)
Name the biggest driver: "add roof orientation to tighten it"
Sensitivity = re-run on ± shocks to the 3 riskiest inputs
